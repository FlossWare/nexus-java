# PR Verification Workflow - Quick Start

**Automatically verify all open pull requests with Claude Code in 3 minutes.**

## What is PR Verify?

**PR Verify** is a Claude Code workflow that automatically:
- ✅ Discovers all open pull requests
- ✅ Verifies each PR in isolation (separate worktrees)
- ✅ Builds the project (Maven + Gradle)
- ✅ Runs all tests (desktop + Android)
- ✅ Checks code quality (Spotless + Checkstyle)
- ✅ Posts verification results as PR comments
- ✅ Provides recommendations (approve / request-changes / needs-review)

**Result:** Know the quality of every open PR without manually checking out each one.

---

## Prerequisites

Your project should have:
- ✅ Automated build system (Maven, Gradle)
- ✅ Test suite (unit + integration)
- ✅ Code quality tools (Spotless, Checkstyle, etc.)
- ✅ GitHub CLI configured (`gh auth login`)

## Quick Start

### Option 1: Run the Pre-Built Workflow

```bash
# In Claude Code CLI or chat
"Run the pr-verify workflow"
```

Or explicitly:

```bash
claude-code workflow run .claude/workflows/pr-verify.js
```

**That's it!** The workflow will:
1. Discover all open PRs
2. Verify each in parallel (isolated worktrees)
3. Post results as comments on each PR
4. Display summary

Claude will use the workflow pattern but customize for your specific needs.

## What Gets Verified?

For each PR, the workflow checks:

### 1. Build Status ✅
- **Desktop**: `./mvnw clean package -DskipTests`
- **Android** (if exists): `gradle :jnexus-android:assembleDebug`
- **Result**: PASSED / FAILED

### 2. Test Status ✅
- **Desktop**: `./mvnw test`
- **Android**: `gradle :jnexus-android:testDebugUnitTest`
- **Metrics**: Number passing, number failing
- **Result**: PASSED / FAILED

### 3. Quality Checks ✅
- **Formatting**: `./mvnw spotless:check`
- **Linting**: `./mvnw checkstyle:check`
- **Violations**: List of formatting/style issues
- **Result**: PASSED / FAILED

### 4. Overall Recommendation
- ✅ **APPROVE**: All checks passed
- ❌ **REQUEST CHANGES**: Build/tests failed
- ⚠️ **NEEDS REVIEW**: Quality issues or mixed results

## Customization

### Verify Only Specific PRs

```javascript
// Pass PR numbers as args
claude-code workflow run .claude/workflows/pr-verify.js --args '[42, 45]'
```

Or modify the workflow script to filter by label:

```javascript
// Filter PRs by label (e.g., "ready-for-review")
const prList = await agent(
  'List open PRs with label "ready-for-review"',
  { schema: PR_SCHEMA }
)
```

### Add Security Scanning

Edit `.claude/workflows/pr-verify.js` to add a security phase:

```javascript
phase('Security Scan')
const securityResults = await agent(
  'Scan PR changes for OWASP Top 10 vulnerabilities',
  { schema: SECURITY_SCHEMA }
)
```

### Skip PR Comments (Just Display Results)

Remove the "Report" phase in the workflow to skip posting comments:

```javascript
// Comment out the entire "Phase 3: Report" section
// Just return the verification results
return { totalPRs, verified, passed, failed, results: validVerifications }
```

## Integration with CI/CD

### GitHub Actions Integration

Create `.github/workflows/pr-verify.yml`:

```yaml
name: PR Verification

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Install Claude Code
        run: curl -fsSL https://claude.ai/install.sh | sh
        
      - name: Verify PR
        run: |
          claude-code workflow run .claude/workflows/pr-verify.js \
            --args '[${{ github.event.pull_request.number }}]'
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### Scheduled Daily Verification

Run verification on all open PRs daily:

```yaml
on:
  schedule:
    - cron: '0 9 * * *'  # 9am daily

jobs:
  verify-all:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: claude-code workflow run .claude/workflows/pr-verify.js
```

## Troubleshooting

### Problem: PRs not found
**Fix:** Ensure GitHub CLI is authenticated: `gh auth login`

### Problem: Worktree conflicts
**Fix:** Clean up stale worktrees: `git worktree prune`

### Problem: Build failures in worktree
**Fix:** Ensure worktree has clean state. Workflow auto-cleans on success.

### Problem: Comments not posting
**Fix:** Check GitHub token permissions: `gh auth status`

## Advanced: Parallel vs Sequential

By default, PRs are verified in **parallel** (faster, uses more resources).

For sequential verification (safer, slower):

Edit the workflow to use `pipeline` instead of `parallel`:

```javascript
// Change from:
const verifications = await pipeline(prList.prs, pr => agent(...))

// To sequential:
const verifications = []
for (const pr of prList.prs) {
  const result = await agent(...)
  verifications.push(result)
}
```
