# JNexus Android - Implementation Guide

Back to [Main Documentation](CLAUDE.md)

## Overview

The Android module (`jnexus-android/`) provides a native mobile app for Android 8.0+ with a modern Jetpack Compose UI. It shares the business logic from jnexus-core but uses platform-specific implementations for HTTP (OkHttp) and storage (EncryptedSharedPreferences).

## Technology Stack

**Core Technologies:**
- **Kotlin**: Primary language for UI and Android-specific code
- **Java 11**: Shared code from jnexus-core (desugared to Android 8.0+)
- **Gradle**: Build system
- **Android SDK 34, minSdk 26** (Android 8.0+)

**UI Technologies:**
- **Jetpack Compose**: Modern declarative UI framework
- **Material Design 3**: Android design system
- **Kotlin Coroutines**: Async/concurrent operations

**Platform Technologies:**
- **OkHttp**: HTTP client (replaces java.net.http for Android compatibility)
- **EncryptedSharedPreferences**: Secure credential storage (AES256_GCM)
- **Desugaring**: Backport Java 11 features (records) to Android 8.0+
- **Logback-Android**: SLF4J implementation for Android

**Dependencies:**
- jnexus-core module (shared business logic)
- Jackson (JSON processing)
- org.json (JSON manipulation)
- SLF4J (logging API)
- JUnit 5 + Mockito (unit testing)
- AndroidX Test (instrumented testing)

## Android Module Components

### Core Components

- **MainActivity.kt**: Main activity with bottom navigation (4 tabs)
- **RepositoryListScreen.kt**: List/delete components with caching
- **SearchScreen.kt**: Advanced filtering interface
- **StatsScreen.kt**: Repository statistics with analytics
- **SettingsScreen.kt**: Credential and configuration management
- **NexusClientOkHttp.java**: HTTP communication, pagination, JSON parsing, metadata extraction (OkHttp)
- **CredentialsAndroid.java**: Encrypted credential storage (EncryptedSharedPreferences)
- **NexusApplication.java**: Application class for dependency injection

### Layered Architecture

```
UI Layer (Jetpack Compose screens: List, Search, Stats, Settings)
    ↓
Service Layer (NexusService.java from jnexus-core)
    ↓
Client Layer (NexusClientOkHttp.java implements NexusHttpClient)
    ↓
HTTP/Nexus API (OkHttp)
```

## HTTP Client (NexusClientOkHttp.java)

Mirrors desktop NexusClient.java functionality using OkHttp instead of java.net.http.

### Key Differences from Desktop

- Uses `OkHttpClient.Builder()` instead of `HttpClient.newBuilder()`
- Uses `Request.Builder()` and `Response` instead of `HttpRequest` and `HttpResponse`
- Implements retry logic via `Interceptor` instead of manual retry loop
- Same caching structure (ConcurrentHashMap with CacheEntry records)
- Same pagination handling (continuation tokens)
- Same JSON parsing (Jackson + org.json)

### Retry Interceptor

```java
private static class RetryInterceptor implements Interceptor {
    private final int maxRetries;
    private final long initialDelayMs;
    
    @Override
    public Response intercept(Chain chain) throws IOException {
        // Exponential backoff: 1s, 2s, 4s
        // Retries on: 5xx, 408, 429, connection errors, timeouts
    }
}
```

### Cache Implementation

- Same 5-minute TTL as desktop
- Same thread-safe ConcurrentHashMap
- Separate caches for RepoRecord and ComponentMetadata
- `isCached()` and `getCacheAge()` helper methods

### Methods

- `listComponents(repository)`: List basic component records (cached)
- `listComponents(repository, forceRefresh)`: Force fresh fetch (bypass cache)
- `listComponentsWithMetadata(repository)`: List components with full metadata (cached)
- `deleteComponent(repository, componentId)`: Delete a component (always fresh)
- `clearCache()`: Clear all caches
- `clearCache(repository)`: Clear cache for specific repository

## Credential Storage (CredentialsAndroid.java)

Uses Android's secure storage instead of file-based.

### Encryption

- **Master key**: Generated by Android Keystore with AES256_GCM scheme
- **Key encryption**: AES256_SIV (deterministic encryption for keys)
- **Value encryption**: AES256_GCM (authenticated encryption for values)
- **Storage**: Private app directory (inaccessible to other apps)

### Implementation

```java
MasterKey masterKey = new MasterKey.Builder(context)
    .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
    .build();

SharedPreferences prefs = EncryptedSharedPreferences.create(
    context,
    "nexus_credentials",
    masterKey,
    EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
    EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
);
```

### Methods

**Standard Credentials methods:**
- `getUrl()`: Get Nexus server URL
- `getUser()`: Get username
- `getPassword()`: Get password
- `getRepositories()`: Get list of configured repositories
- `getDefaultRepository()`: Get default repository for UI pre-population
- `getDefaultRegex()`: Get default regex filter
- `getDefaultDryRun()`: Get default dry-run setting
- `getHttpTimeoutSeconds()`: Get HTTP timeout configuration

**Android-specific methods:**
- `hasCredentials()`: Check if credentials configured
- `clearAll()`: Remove all stored data
- `saveCredentials(url, user, password)`: Save credentials
- `saveRepositories(repositories)`: Save repository list
- `saveDefaults(repository, regex, dryRun)`: Save UI defaults

## Jetpack Compose UI Screens

### RepositoryListScreen.kt

List and delete components with caching.

**Features:**
- Repository dropdown (uses `ExposedDropdownMenuBox`)
- List/Refresh buttons (coroutines with Dispatchers.IO)
- LazyColumn for component cards
- Delete confirmation dialogs
- Component details dialog (AlertDialog with metadata)
- Error handling with Card + error color scheme

**Implementation Pattern:**
```kotlin
val scope = rememberCoroutineScope()
Button(onClick = {
    scope.launch {
        loading = true
        try {
            withContext(Dispatchers.IO) {
                components = service.getRepositoryRecords(repository, forceRefresh = true)
            }
        } catch (e: Exception) {
            error = e.message
        } finally {
            loading = false
        }
    }
})
```

## Dependency Injection (NexusApplication.java)

Application class provides singleton instances.

**Pattern:**
```java
public class NexusApplication extends Application {
    private static NexusClientOkHttp nexusClient;
    private static CredentialsAndroid credentials;
    private static NexusService nexusService;
    
    @Override
    public void onCreate() {
        super.onCreate();
        credentials = new CredentialsAndroid(this);
        nexusClient = new NexusClientOkHttp(
            credentials.getUrl(),
            credentials.getUser(),
            credentials.getPassword(),
            credentials.getHttpTimeoutSeconds()
        );
        nexusService = new NexusService(nexusClient);
    }
    
    public static NexusService getNexusService() {
        return nexusService;
    }
    
    public static CredentialsAndroid getCredentials() {
        return credentials;
    }
}
```

## Testing

### Unit Tests (NexusClientOkHttpTest.java)

- Uses OkHttp MockWebServer for HTTP mocking
- Tests: pagination, caching, retry logic, metadata parsing, error handling
- JUnit 5 + Mockito
- 13 tests covering all NexusHttpClient methods

**Test Pattern:**
```java
@Test
void testListComponents_withPagination() {
    mockWebServer.enqueue(new MockResponse()
        .setResponseCode(200)
        .setBody(page1Json));
    mockWebServer.enqueue(new MockResponse()
        .setResponseCode(200)
        .setBody(page2Json));
    
    List<RepoRecord> records = client.listComponents("maven-releases");
    
    assertEquals(20, records.size()); // 10 per page
    assertEquals(2, mockWebServer.getRequestCount());
}
```

### Instrumented Tests (CredentialsAndroidTest.kt)

- Requires Android runtime for EncryptedSharedPreferences
- Tests: save/get credentials, repository parsing, encryption, persistence
- AndroidX Test framework
- 15 tests covering all Credentials methods

**Test Pattern:**
```kotlin
@RunWith(AndroidJUnit4::class)
class CredentialsAndroidTest {
    private lateinit var context: Context
    private lateinit var credentials: CredentialsAndroid
    
    @Before
    fun setUp() {
        context = ApplicationProvider.getApplicationContext()
        credentials = CredentialsAndroid(context)
        credentials.clearAll() // Clean state
    }
    
    @Test
    fun testSaveAndGetCredentials() {
        credentials.saveCredentials(
            "https://nexus.example.com",
            "admin",
            "secret"
        )
        
        assertEquals("https://nexus.example.com", credentials.getUrl())
        assertEquals("admin", credentials.getUser())
        assertEquals("secret", credentials.getPassword())
    }
}
```

**Key points:**

```gradle

    compileSdk 34

        minSdk 26  // Android 8.0+
```

**Key points:**

## Building and Running

### Build Debug APK

```bash
gradle :jnexus-android:assembleDebug
```

**Output**: `jnexus-android/build/outputs/apk/debug/jnexus-android-debug.apk`

### Build Release APK

```bash
gradle :jnexus-android:assembleRelease
```

**Output**: `jnexus-android/build/outputs/apk/release/jnexus-android-release-unsigned.apk`

**Note**: Requires signing configuration in `gradle.properties` or environment variables.

### Run Unit Tests

```bash
gradle :jnexus-android:testDebugUnitTest
```

### Run Instrumented Tests

```bash
gradle :jnexus-android:connectedDebugAndroidTest
```

**Note**: Requires connected Android device or emulator.

### Install on Device

```bash
adb install jnexus-android/build/outputs/apk/debug/jnexus-android-debug.apk
```

## Common Development Tasks

### Adding a New Screen

1. Create new Kotlin file in `ui/screens/` package
2. Define `@Composable` function for the screen
3. Add navigation in `MainActivity.kt`
4. Update bottom navigation bar if needed
5. Test on multiple screen sizes (phone, tablet)
6. Update README.md and CHANGELOG.md

### Adding a New Feature to Settings

1. Add UI field to `SettingsScreen.kt`
2. Add storage method to `CredentialsAndroid.java`
3. Update `NexusApplication.java` if DI changes needed
4. Add tests in `CredentialsAndroidTest.kt`
5. Update CHANGELOG.md

### Modifying HTTP Client

1. Update `NexusClientOkHttp.java`
2. Keep interface compatible with `NexusHttpClient` from jnexus-core
3. Add tests in `NexusClientOkHttpTest.java`
4. Consider desktop compatibility if changing shared interface

## Debugging

### Android Studio Logcat

```bash
adb logcat | grep JNexus
```

### Verbose Logging

Add to service methods:
```java
Log.d("JNexus", "Fetching page with token: " + token);
```

### HTTP Request Inspection

Use OkHttp logging interceptor:
```java
HttpLoggingInterceptor logging = new HttpLoggingInterceptor();
logging.setLevel(HttpLoggingInterceptor.Level.BODY);
client = new OkHttpClient.Builder()
    .addInterceptor(logging)
    .build();
```

### Encrypted SharedPreferences Inspection

Cannot directly view encrypted values. To debug:
```java
Log.d("JNexus", "Stored URL: " + credentials.getUrl());
```
